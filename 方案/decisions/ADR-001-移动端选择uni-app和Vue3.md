# ADR-001-移动端选择 uni-app 和 Vue 3

状态：Accepted  
日期：2026-06-03

## 背景

BlueNote 主要面向移动端用户，需要实现登录、浏览笔记、发布笔记、评论、点赞收藏、关注、Feed、IM 等功能。

项目当前只设计移动端和后端，不优先建设 PC Web。移动端需要兼顾开发效率、学习成本、后续多端扩展能力和真实项目可维护性。

## 备选方案

| 方案 | 说明 |
|---|---|
| 原生 Android + iOS | 体验最好，但需要双端开发，成本最高 |
| Flutter | 跨端体验较好，但需要 Dart 技术栈 |
| React Native | 跨端能力强，但生态和工程复杂度较高 |
| uni-app + Vue 3 | 基于 Vue 生态，适合移动端和多端发布 |
| Vue H5 + WebView | 简单，但 App 能力和体验边界较弱 |

## 决策

移动端第一阶段选择：

```text
uni-app + Vue 3 + TypeScript + Pinia
```

目标平台优先支持：

1. Android
2. iOS
3. H5

后续可根据需要扩展小程序。

## 决策原因

1. Vue 生态学习成本较低，适合快速推进项目。
2. uni-app 面向移动端和多端发布，比单纯 H5 包壳更合适。
3. TypeScript 可以提升中长期维护质量。
4. Pinia 是 Vue 3 生态下更自然的状态管理方案。
5. BlueNote 当前不是强 3D、重动画、复杂原生能力项目，不必一开始选择原生双端。

## 影响

正向影响：

1. 移动端开发效率较高。
2. 可以复用 Vue 生态。
3. 后续多端扩展空间较好。
4. 初期人力成本可控。

代价：

1. 极致原生体验不如原生 Android/iOS。
2. 复杂原生能力可能需要插件或原生扩展。
3. 工程规范需要从一开始约束，否则多端兼容会变复杂。

## 后续重新评估条件

出现以下情况时重新评估：

1. App 对复杂原生能力依赖明显增加。
2. 性能或体验无法满足核心业务。
3. 团队具备稳定原生双端开发能力。
4. 后续需要重度动画、音视频编辑、复杂图形处理等能力。
